# 프로젝트 기획서 — Novel-pick

FE 취업용 포트폴리오 프로젝트. AI 에이전트 활용 능력 증명이 핵심 목적
(도치팀 사전 과제 아이디어에서 착안, 실제 지원과 무관한 자율 프로젝트).

## 서비스 개요

**웹소설 리뷰·추천 플랫폼 + 실시간 댓글 + AI 대화형 추천 에이전트**

- 기존 서비스 `소설넷(ssn.so)`이 사실상 유일한 경쟁자이며, 사용자가 실제 관심 있는 취미(웹소설) 도메인
- 소설넷에는 정적인 "AI 리뷰 요약" 기능은 있지만, 대화형으로 질의에 응답하는 에이전트는 없음 → 이 대화형 추천 레이어가 핵심 차별점
- 작품별 실시간 댓글 기능 추가로 커뮤니티성 보완

## 핵심 시나리오

> 사용자는 웹소설 작품 상세 페이지에서 실시간으로 갱신되는 후기·댓글을 읽고 남기며,
> AI 에이전트에게 대화형으로 "이 작품과 비슷한 소설 추천해줘", "명작 리스트 보여줘" 같은
> 질문을 던져 다음 읽을 작품을 찾는다.

## 차별점 (vs 소설넷)

| 항목 | 소설넷(ssn.so) | Novel-pick |
| --- | --- | --- |
| AI 리뷰 요약 | 정적 요약 제공 | 대화형 추천 에이전트 (질의응답) |
| 댓글 | - | 작품별 실시간 댓글 (Socket.IO) |
| 추천 방식 | 없음 | "비슷한 작품", "명작 리스트" 등 자연어 질의 |

## 주요 기능 목록

1. 작품 목록/상세 페이지 (메타데이터, 표지, 리뷰·평점)
2. 작품별 실시간 댓글 (Socket.IO room 단위)
3. AI 대화형 추천 에이전트 (유사 작품 추천, 명작 리스트 등 자연어 질의)
4. 장르별/플랫폼별 트렌드 및 작품 간 태그·유사도 네트워크 시각화
5. 모니터링 대시보드 (소켓 재연결 실패율, AI 응답 지연, 렌더 성능)

## 기술 스택 & 요구사항 매핑

| 요구 기술 | 충족 방식 |
| --- | --- |
| Socket.IO 실시간 데이터 처리 | 외부 실시간 API가 없는 도메인이므로, 작품별 실시간 댓글을 자체 구축(room 단위)하여 충족 |
| 대용량 신호 데이터 시각화 (Babylon.js/R3F/d3/Konva) | 필수 요건을 축소 적용 — 장르별/플랫폼별 트렌드 네트워크 그래프, 작품 간 태그·유사도 네트워크 등 경량 시각화로 대체 (진짜 "대용량 신호 데이터"는 아님을 인지) |
| Datadog/Sentry/PostHog | 소켓 재연결 실패율, AI 에이전트 응답 지연, 렌더 성능 모니터링 |

## 데이터 소스 및 정책

- **표지 이미지**: ssn.so와 동일한 방식으로 네이버/카카오 CDN 이미지 URL을 직접 핫링크 + 각 작품 카드에 "원본에서 보기" 출처 링크 부착. 서버에 재호스팅하지 않음. 법적으로 100% 안전하진 않지만 이미 운영 중인 경쟁 서비스와 동일한 리스크 수준으로 판단, 감수하고 진행
- **텍스트 메타데이터(제목/작가/장르/줄거리)**: 네이버시리즈/카카오페이지 공개 목록·상세 페이지 크롤링
  - ⚠️ **미결정 사항**: 줄거리 원문을 그대로 복사하면 저작권 이슈 소지 있음 → 화면/아키텍처 설계 단계에서 표시 방식(원문 그대로 vs AI 재요약) 결정 필요
- **리뷰/평점/댓글**: 자체 구축(사용자 생성 콘텐츠)이므로 외부 API 법적 이슈 없음
- **도서 검색 API(카카오/네이버)**: ISBN 기반이라 연재 웹소설 커버 불가 확인됨 → 크롤링 방식 채택 근거
